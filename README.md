# Allin-One

> 每天在 RSS、B站、YouTube、技术博客、新闻网站之间来回切换？花大量时间阅读，却只吸收了一小部分？

**Allin-One 是你的私人信息操作系统。** 把散落在各个平台的信息自动汇集到一处，让 AI 帮你阅读、分析、整理——你只需要看结论、做决策。

支持 1,000+ 订阅源同时监控，每日自动处理约 2,000 条内容，从采集到分析完成不超过 5 分钟。部署在你自己的服务器上，数据完全属于你。

---

## 它能帮你做什么

### 所有信息，一处阅读

不再切换十几个浏览器标签。RSS 订阅、B站UP主更新、YouTube 频道、播客节目、技术博客、甚至宏观经济数据——全部汇入同一个信息流，像刷朋友圈一样浏览。已有 RSS 订阅？OPML 一键导入，迁移零成本。

### AI 替你读，你只看结论

每条内容自动经过 AI 分析：一句话摘要、关键观点提取、情感倾向判断、相关人物/时间/地点识别。30 秒扫完 10 篇文章的核心信息，遇到感兴趣的再展开深读。你还可以自定义提示词模板，让 AI 按你关心的角度去分析。

### 视频也不放过

关注的B站UP主发了新视频？YouTube 频道更新了？系统自动下载到本地，内嵌播放器直接看，不用跳转到第三方平台。字幕自动提取，AI 同样可以分析视频内容。

### 你的规则，你做主

不同内容用不同处理方式——这是 Allin-One 最灵活的地方：

- 英文科技博客 → 自动翻译成中文 → AI 生成摘要
- B站UP主视频 → 自动下载 → 提取字幕 → AI 分析要点
- 宏观经济数据 → 自动采集 → AI 生成趋势分析报告
- 中文新闻源 → 直接 AI 分析 → 推送到钉钉

每个数据源绑定一条处理流水线，步骤自由组合，一次配置，长期自动运行。

### 越用越聪明的调度

系统会自动学习每个订阅源的更新规律：日更的源每小时检查，周更的源每天检查一次，长期不更新的自动降频。采集失败时智能退避重试，无需人工干预。你也可以手动设定固定频率，或随时一键触发。

### 随时追问 AI

对任何一条内容，你都可以发起 AI 对话：「这篇文章的论据充分吗？」「帮我用通俗语言解释下这个概念」「跟上周那条新闻有什么关联？」——信息消费不再是单向的被动接收。

### 重要的，离线留存

一键收藏值得深入的内容。收藏的视频自动下载到本地，即使原始链接失效也能随时回看。独立的媒体管理页面让你集中浏览所有离线视频和音频。

### 主动推送，不遗漏

配置好推送规则后，AI 分析完的重要信息可以自动发到钉钉、邮件或自定义 Webhook。不用守着平台等更新，信息会主动找到你。

## 典型使用场景

| 我想要... | 怎么配 | 效果 |
|-----------|--------|------|
| 跟踪B站UP主的视频更新 | 添加 RSSHub 数据源 + "视频下载分析"流水线 | 新视频自动下载、提取字幕、AI 生成内容摘要 |
| 阅读英文科技博客不费劲 | 添加 RSS 订阅 + "英文文章翻译分析"流水线 | 文章自动翻译成中文，附带 AI 分析 |
| 监控某个网页的更新 | 添加网页抓取数据源 + "文章分析"流水线 | 网页变化时自动抓取、AI 生成变更摘要 |
| 关注宏观经济指标 | 添加 AkShare 数据源 + "金融数据分析"流水线 | 定期采集 CPI 等指标，AI 生成趋势分析 |
| YouTube 频道自动翻译 | 添加 RSSHub 数据源 + "视频翻译分析"流水线 | 新视频下载、字幕翻译、AI 分析要点 |
| 纯粹当 RSS 阅读器用 | 添加 RSS 订阅，不绑定流水线 | 仅采集展示，不做 AI 处理 |

## 和其他工具有什么不同

| | Allin-One | RSS 阅读器 (Feedly/Miniflux) | 稍后读 (Readwise/Pocket) |
|---|---|---|---|
| 信息采集 | RSS + B站 + YouTube + 播客 + 网页 + 金融数据 | 仅 RSS | 手动保存 |
| AI 分析 | 自动分析每条内容，可自定义提示词 | 无 | 有限的高亮/摘要 |
| 视频处理 | 自动下载、字幕提取、AI 分析 | 无 | 无 |
| 处理流程 | 自定义流水线，步骤自由编排 | 固定 | 固定 |
| 数据所有权 | 自部署，数据在你手里 | 云服务，数据在别人手里 | 云服务 |
| 主动推送 | 钉钉/邮件/Webhook | 无或有限 | 无 |

## 三步开始

**第一步：部署**

一台普通的 Linux 服务器、NAS 或 VPS（2 核 4G 够用），装好 Docker 即可。

```bash
git clone <repo-url> && cd allin-one
cp backend/.env.example backend/.env
# 编辑 .env，填入你的大模型 API Key（支持 OpenAI 兼容接口）
docker compose up -d --build
```

**第二步：添加数据源**

打开浏览器访问 `http://your-server:8000`，在「数据源管理」页面添加你想关注的订阅源。已有 RSS 订阅？直接导入 OPML 文件。

**第三步：查看信息流**

切换到「信息流」页面，你的内容已经开始自动采集和分析了。喝杯咖啡，然后回来看 AI 帮你整理好的信息摘要。

## 产品模块一览

| 模块 | 做什么 |
|------|--------|
| **信息流** | 你的主阅读界面。卡片式浏览，每条内容附带 AI 分析摘要，支持按数据源筛选、收藏、展开原文 |
| **数据源管理** | 管理所有订阅。支持 10 种数据源类型，OPML 批量导入导出，一键触发全部采集 |
| **内容库** | 所有采集内容的表格视图。多条件筛选、排序、去重，适合批量管理和检索 |
| **流水线** | 配置和监控处理流程。可视化编辑步骤组合，查看每次执行的详细日志 |
| **媒体管理** | 集中管理所有下载的视频和音频。在线播放，查看元数据 |
| **收藏** | 值得深入的内容专属视图。收藏可自动触发视频下载 |
| **金融数据** | 宏观经济指标的专属展示页面 |
| **仪表盘** | 系统运行状态一目了然：采集统计、执行记录、异常告警 |
| **系统设置** | 大模型 API 配置、数据保留策略、通知渠道、凭证管理 |

## 版本演进

**v1.0 — 从零到能用**
